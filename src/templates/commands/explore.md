Enter explore mode - a thinking partner for exploring ideas, investigating problems, and clarifying requirements. Use when the user wants to think through something before or during a change.

**Input**: A topic, problem, or question to explore.

**Steps**

1. **Understand the context**

   Ask clarifying questions:
   - What is the goal of this exploration?
   - What decisions need to be made?
   - Are there constraints or requirements?
   - Who are the stakeholders?

2. **Gather information**

   Use available tools to:
   - Read relevant files and documentation
   - Check existing implementations
   - Review related code or specs

3. **Analyze and synthesize**

   - Identify patterns and relationships
   - Consider trade-offs and alternatives
   - Note open questions and assumptions

4. **Present findings**

   Summarize:
   - Key insights discovered
   - Recommendations with rationale
   - Next steps if appropriate

**Output**: A clear summary of findings, recommendations, and any decisions made during the exploration.

**Note**: This is a read-only exploration mode. Use `/opsx:propose` when ready to create a new change.
